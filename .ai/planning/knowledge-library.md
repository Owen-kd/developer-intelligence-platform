# 지식 도서관(Knowledge Library) — 비전 & 로드맵

> 목적: **퇴사로 인한 도메인 지식 퇴화 방지** + 히스토리 파악 + 문의 자동응대.
> 회사 도메인 지식을 정제·분류·축적한 "도서관"을 만들어, 팀 무관 누구나 필요한 데이터를 얻는다.

## 현재 상태 (로컬 POC, LLM 0)
- 수집: 실 Jira(PA20, 페이지네이션/멱등) + Git 멀티repo(백/프론/워커) 링크.
- enrich: 본문(description) · 담당자/문의자 · 서가(components 자동분류).
- 축적: Postgres(로컬 Docker). 예) 이슈 1000 · 본문 987 · 담당자 972 · 링크 156 · knowledge 1000.
- 서가 자동분류(LLM 0): 1:1 문의 · 상품개선/오류 · GSSHOP · 쿠팡 · 결제 …
- 실행: `apps/cli/collect` (수동, 재실행 멱등).

## 데이터 소스 (도서관 장서)
| 소스 | 내용 | 신뢰 | 상태 |
|------|------|:---:|------|
| Jira | 이슈/본문/담당자/서가/코멘트 | 중 | ✅ |
| Git | 커밋/링크(멀티repo) | 중 | ✅ (PR/코드파일 enrich 예정) |
| 운영팀 답변(CS) | 문의↔답변 | 중~상 | ⏳ 위치 확인 필요 |
| 전문가 정리 문서 | 검증된 지식 | **상** | ⏳ 위치 확인 필요 |

## 단계별 로드맵
- **Phase 1 — 재료 완성 (지금, 로컬, LLM 0)**: 수집·enrich·서가 ✅ → **제공(검색) 얇게** → POC 검증.
- **Phase 2 — 지식 승격 (LLM)**: 노이즈필터 + LLM 위키(문제/원인/해결) + 벡터검색 + `/ask` RAG.
- **Phase 3 — 배포/자동화**: 서버·클라우드 호스팅 + 스케줄러/cron(증분) + 팀 접속.
- **Phase 4 — 확장**: 운영답변·전문가문서 소스 추가, 서가 정교화, 역할별 뷰(개발/운영).

## 👤 사용자가 앞으로 할 일 (내가 못 하는 것)

### A. 지금 당장 (보안)
- [ ] `.env` 의 실 **Anthropic 키 · Jira 토큰 재발급(rotate)** — 대화에 노출됨.

### B. 방향 결정 (필요할 때)
- [ ] **배포 위치**: 로컬 검증 후 → AWS? 사내서버? (지금은 미정 OK)
- [ ] **우선 타겟**: 어느 팀/유스케이스부터? (운영 문의응대 / 개발 히스토리 …)
- [ ] **LLM 켤 시점**: 위키·요약·RAG 하려면 실 LLM 필요(비용). 언제 켤지.

### C. 데이터 소스 제공 (있으면)
- [ ] **운영팀 답변** 위치 (채널톡/Zendesk/이메일/Jira코멘트/스프레드시트?)
- [ ] **전문가 정리 문서** 위치 (Confluence/Notion/구글독스?) — 없으면 "앞으로 어디에 쓸지"
- [ ] Jira 다른 프로젝트(ENG 등)도 수집할지

### D. 검증/피드백 (POC 평가)
- [ ] 뇌 답변 품질 보고 "쓸만한지" 판단 → 방향 조정
- [ ] 서가(components 기반 분류)가 적절한지 확인

## 🤖 빌드 측이 할 일 (참고)
제공(검색/질의) 얇게 → PR/코드파일 enrich → 노이즈필터+LLM 위키 → 벡터검색+/ask →
증분커서+스케줄러(배포 대비) → 운영답변/전문가문서 커넥터 → 배포.
