# APR-002 — Jira 접근 · 자격증명 · PII 정책

- 상태: Pending Approval
- 요청일: 2026-07-07
- 요청자: Project Planner (AI)
- 관련 Sprint: Sprint-03 (Jira Collector)

## 요청 요약
실제 Jira에서 이슈/코멘트를 수집하기 위한 **접근 범위·자격증명 관리·개인정보(PII) 취급** 정책을 승인받는다.

## 왜 사람이 결정해야 하나
- 실데이터/자격증명/개인정보는 보안·법적 책임이 따른다. AI가 임의로 정할 수 없다.
- system.md: 비밀키는 `.env`(gitignore), `.env.example`만 커밋.

## 결정 필요 항목
1. **접근 범위**: 어떤 프로젝트/이슈타입까지 수집? 읽기 전용 확정?
2. **자격증명**: 저장 위치(로컬 `.env` → 후일 시크릿 매니저), 토큰 발급 주체.
3. **PII**: 코멘트/작성자 개인정보 저장 여부·마스킹·보존기간.
4. **대상 인스턴스**: 실서비스 Jira vs 샌드박스(초기엔 샌드박스 권장).

## 선택지
1. **(추천)** 샌드박스/테스트 프로젝트 + 읽기 전용 + 작성자 식별자 최소화로 시작.
2. 실 Jira 읽기 전용, PII 마스킹 정책 동반.
3. 보류 — 고정 픽스처(fixture)로 개발, 실연동은 후속.

## 영향
- 승인 시: Sprint-03가 실제/샌드박스 연동으로 진행.
- 미승인 시: Sprint-03는 목/픽스처로만 진행(DoD 일부는 통합 단계로 이연).
