# APR-009 — 인증 모델 / 역할 범위

- 상태: Pending Approval
- 요청일: 2026-07-07
- 요청자: Project Planner (AI)
- 관련 Sprint: Sprint-12 (Auth + Audit)
- 관련 ADR: ADR-NNN (제안)

## 요청 요약
`platform/auth`가 구현할 **인증 방식과 권한 범위**를 확정한다(내부 도구 수준 vs 멀티테넌트).

## 왜 사람이 결정해야 하나
- 인증/권한 모델은 보안·조직 정책·데이터 접근 경계를 정한다. 되돌리기 비싸다.

## 결정 필요 항목
1. **인증 방식**: 사내 SSO/OIDC · API 토큰 · 세션 중 무엇.
2. **테넌시**: 단일 조직(내부 도구) vs 멀티테넌트(조직별 격리).
3. **권한 모델**: 단순 역할(뷰어/운영자) vs 세밀 RBAC.

## 선택지
1. **(추천)** 내부 도구 전제: 단일 조직 + API 토큰/사내 SSO + 최소 역할(뷰어/운영자).
2. 멀티테넌트 대비 설계(조직 격리 스키마) — 범위 증가.
3. 보류 — Sprint-11까지 무인증(내부망), Sprint-12 착수 시 결정.

## 영향
- 승인 시: Sprint-12가 선택 모델로 auth 구현, 리포트 라우트 보호.
- 미승인 시: Sprint-12 착수 불가 또는 무인증 유지(노출 리스크).
