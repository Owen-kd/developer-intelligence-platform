# APR-007 — EventBus 브로커 백엔드 전환 시점

- 상태: Pending Approval
- 요청일: 2026-07-07
- 요청자: Project Planner (AI)
- 관련 Sprint: (스케일 시점) — 기본은 Sprint-02 in-memory 유지
- 관련 ADR: ADR-003 후속

## 요청 요약
EventBus는 [ADR-003]대로 **초기 in-memory**로 시작한다. 언제 **Redis/브로커 백엔드**로 전환할지의 **트리거 기준**을 사전 합의한다.

## 왜 사람이 결정해야 하나
- 브로커 전환은 배포 토폴로지·내구성·운영을 바꾸는 결정.
- 조기 전환은 과설계(YAGNI 위반), 지연은 신뢰성 리스크.

## 선택지
1. **(추천)** in-memory 유지 + 전환 트리거 명문화:
   - 프로세스(api/worker/scheduler) 간 **크로스 프로세스 이벤트**가 필요해질 때, 또는
   - 이벤트 유실이 허용 불가(내구성/재처리) 해질 때.
2. Phase 3(Agent 비동기 실행) 시작 시점에 선제 전환.
3. 미정 — 필요할 때 그때 ADR.

## 영향
- 승인 시: 인터페이스는 지금부터 교체 가능하게 유지(Sprint-02에서 이미 추상화), 실제 전환은 트리거 충족 시 별도 Sprint.
- 미승인 시: 전환 판단 기준 부재로 스케일 시 임기응변.
