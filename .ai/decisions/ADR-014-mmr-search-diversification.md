# ADR-014 — MMR 검색 다양화 (같은 주제 중복 노출 억제)

- 상태: Accepted
- 날짜: 2026-07-10
- 관련: [ADR-013](ADR-013-cross-encoder-reranker.md)(리랭커) · [ADR-009](ADR-009-local-embedding-pgvector.md)(임베딩) · 하이브리드 검색(fusion.py)

## 맥락
검색 top-k 가 **같은 주제 계열로 도배**된다. "쿠팡에서 옵션이 수정이 안돼요" 질의의 top-5 가
전부 "쿠팡 옵션 수정" 변형이라, 사용자가 서로 다른 관련 사안(추천옵션 정책·GTIN 속성 등)을 못 본다.
정규화(canonicalization) 방향을 검토하며 측정한 결과: 59개 위키 중 **코사인 ≥0.90 쌍이 470개**.
같은 도메인이라 임베딩 바닥 유사도가 높다 — 진짜 "중복"이 아니라 "같은 주제 계열"이다.
→ 파괴적 병합/클러스터링은 서로 다른 이슈를 뭉갠다. **비파괴 검색 다양화**를 택한다(오너 결정).

## 결정
retrieve → (리랭커) → **MMR(Maximal Marginal Relevance) 로 top-k 선정**. 후보 풀(rerank_pool, 기본 20)에서
관련도와 다양성을 절충해 고른다. 순수 함수 `modules/knowledge/application/diversify.py::mmr_select`.

- **관련도 = 입력 순위(랭크)** — 리랭커 로짓/융합 코사인 등 서로 다른 척도에 안전.
- **다양성 = 위키 임베딩 코사인, 단 풀 안에서 min-max 정규화(핵심)**.
  절대 코사인이 0.899~0.925(스프레드 ~0.03)로 몰려 있어 **절대값 페널티는 거의 상수 → 재정렬 실패**
  (λ=0.7~0.3 에서 무변화, λ=0.15 라야 움직이나 그땐 관련도가 붕괴). 그래서 각 선택 단계에서 남은
  후보의 max_sim 을 min-max 정규화해 "가장 겹치는 후보=1.0, 가장 안 겹치는=0.0" 으로 상대화한다.
- 설정: `diversify_enabled`(기본 ON) · `diversify_lambda`(0.7, 1.0=순수 관련도 / 0.0=순수 다양성).
- 배선: `wiki_pipeline.hybrid_search`(/ask) · MCP `search_wiki`. gap 판정은 벡터 커버리지 유지(불변).

## 근거
- MMR 은 정보검색 표준 다양화 기법 — 관련도를 크게 해치지 않으며 상위 중복만 억제.
- 풀-상대 정규화로 "단일 도메인의 균일한 유사도" 문제를 정면 해결(관련도는 랭크로, 다양성은 정규화로 — 대칭).
- 비파괴: 원본/위키 불변, 검색 순간만 재정렬. `diversify_enabled=False` 로 즉시 롤백.

## 절충 / 리스크 (정직 보고)
- 풀-상대 정규화는 **항상 최대 강도로** 다양성을 적용한다(풀이 진짜 한 주제여도 가장 덜 겹치는
  후보를 밀어올림). λ 로 통제하나, 관련도가 조밀한 풀에선 관련도 낮은 항목이 tail 로 올 수 있다.
  → λ=0.7 로 관련도 우위 유지, top-1 은 라이브에서 보존됨(아래).
- 단일 도메인(현재 상품)에선 다양성 신호 자체가 약함 — 다도메인으로 커지면 절대 유사도 차이가
  벌어져 더 자연히 작동한다. 정규화는 그 전까지의 브리지.
- 임베딩 없는 후보는 sim=0(최대 다양) 취급 — 드물게 미임베딩 위키가 상위로 올 수 있으나 현재 전량 임베딩.

## 결과
- `modules/knowledge/application/diversify.py`(신규, 순수) + `repository.embeddings_for`(후보 임베딩 조회) + `_parse_vector`.
- `shared/config/settings.py`: `diversify_enabled/diversify_lambda`. `wiki_pipeline`·`apps/mcp` 배선.
- 유닛 6건. **라이브 A/B**(리랭커 OFF, "옵션 수정 안돼요"): 다양화 OFF=5행 모두 "쿠팡 옵션 수정" →
  ON(λ0.7)=top-1(PA20-19864) 보존 + tail 이 추천옵션 정책·GTIN 속성 등 **다른 사안으로 확장**. 정밀 질의도 top-1 불변.
