# ADR-015 — 이슈 Facet 택소노미 (분류 우선 구조)

- 상태: **Accepted** (오너 승인 2026-07-10)
- 날짜: 2026-07-10
- 관련: [taxonomy.md](../knowledge/taxonomy.md) · [ADR-010](ADR-010-team-shelf-access-control.md)(서가/접근제어) · [knowledge-lifecycle](../architecture/knowledge-lifecycle.md) · ③ Graphify

## 맥락
현재 이슈는 Jira 원형 그대로 저장되고, 조직 기준은 지저분한 `components`(예: `상품-오류-툴`,
`[1:1문의]버그-내부이슈`)에 의존한다. 오너 요구: **"이슈를 저장만 하지 말고, 어떤 내용에 속하는지
먼저 자른다. 전체 구조를 먼저 잡는다."** 위키 405건+타 도메인을 만들기 전에 분류 골격을 고정해야
소급 재작업이 없다.

핵심 데이터(5,000건): 유형 토큰 오류1,117/개선621/문의3,118 · 도메인 상품401·주문285·재고88 ·
팀 PA20(툴)3,000/ENG(엔진)2,000. `components` 는 이미 `도메인-유형-팀` 을 한 문자열에 압축한 상태다.

## 결정
이슈에 **직교하는 6개 facet 축**(도메인·기능영역·액션·채널·유형·팀/영역)을 부여하는 분류 레이어를 둔다.
단일 깊은 트리가 아니라 facet — "상품>매칭>수정" 경로는 축 조합의 한 뷰로 도출한다.
**통제 어휘는 지어내지 않고 백엔드 실 도메인 문서에 정렬한다**(`gmp.openapi.2023/.ai`:
`domain-map.md` 도메인 레지스트리 · `glossary.md` 마켓코드·용어 · `domains/product/overview.md` 기능영역).
상세 어휘·매핑은 [taxonomy.md]. 분류는 규칙 부트스트랩 + 고정어휘 LLM 보강, `IssueClassified` 이벤트로 적재(원본 불변).

## 대안 비교
1. **단일 계층 트리**(오너 최초 예시 `상품>쇼핑몰>쿠팡>수정`). 직관적이나 다차원 이슈에서 가지 충돌
   (오류vs개선, 다중 도메인) → 중복·경직. 오너도 "중복으로 넣어야" 라며 facet 필요성을 직감.
2. **Facet(채택)**: 축 분리 → 중복 없이 다뷰. 경로는 조합으로. 검색 필터·접근제어·그래프와 정합.
3. **분류 안 함(현행 유지)**: components 그대로. 조직·탐색·gap 분석이 계속 지저분.

## 근거
- 이미 존재하는 `components` 의 압축 택소노미를 **분해**하는 것이라 데이터로 뒷받침됨(추측 아님).
- facet 은 접근제어 서가(도메인/팀), Obsidian 폴더/태그, gap "빈 분류", Graphify 노드로 **재사용**된다 —
  한 번의 분류가 여러 기능을 업그레이드.
- 규칙으로 유형·팀 100%, 나머지는 제한 LLM → 신뢰성 있는 자동화.

## 영향 (아키텍처)
- **비파괴/이벤트 정합**: 이슈 원본 불변, 분류는 새 레이어(`issue_facets` 신규 테이블 초안) + 이벤트.
  헌법("히스토리 덮어쓰기 금지") 준수. 재분류는 새 버전.
- **의존성 방향**: 분류는 `modules/`(application: 규칙 순수 + LLM 포트) — 외부 호출은 infrastructure 경유.
- **기존 ADR-010 서가**: 도메인/팀 facet 이 서가 정의를 대체·정교화(마이그레이션 필요 시 후속 ADR).
- **마이그레이션**: `NNN_issue_facets.sql`(신규 테이블 + 인덱스). 롤백 = 테이블 drop(원본 이슈 영향 0).

## 절충 / 리스크
- 통제 어휘 관리 비용(신규 도메인/채널 등장 시 확장). → 어휘를 config 로 두고 `미상` 폴백.
- LLM 분류 오류 가능 → 고정 어휘 제한 + 스키마 검증 + `미상` 폴백 + (후속) 전문가 검수로 완화.
- 문의(3,118건)는 도메인 토큰이 약해 LLM 의존↑ — 초기엔 개발성 이슈(상품/주문/재고) 우선 분류, 문의는 후속.
- 다축 걸침(도메인 2개) 예외 — 초안은 "주 도메인 1 + 보조 목록".

## 구현안 (승인 후, 단계)
1. 통제 어휘 확정(config) + `issue_facets` 마이그레이션.
2. 규칙 분류기(순수, unit 테스트) → 기존 5,000건 부트스트랩.
3. 고정어휘 LLM 분류기(액션 등) — 프롬프트 `prompts/knowledge/classify.md`.
4. 수집 파이프라인 배선(신규 이슈 자동 분류, `IssueClassified`).
5. 활용: 검색 facet 필터 · Obsidian facet 뷰 · gap 빈분류 리포트.

## 결과 (승인 시)
- 문서: [taxonomy.md] 확정본. 코드: `modules/knowledge`(분류) + 마이그레이션 + 프롬프트.
- 미승인 시: 이 ADR 은 제안으로 남고 구현 착수하지 않음.
