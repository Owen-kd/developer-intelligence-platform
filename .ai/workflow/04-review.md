# 04 — Review (검토)

> 목적: 병합 전에 정확성·아키텍처·품질을 독립 시각으로 검증한다.
> 개발 방식: GPT 설계 → Claude 구현 → **GPT/다른 AI 리뷰**. 구현자와 리뷰어를 분리한다.

## 입력
- 구현된 변경(diff) + 설계/Discovery 노트

## 리뷰 체크리스트
### 정확성
- [ ] 성공 기준(Discovery)을 실제로 충족하는가.
- [ ] 엣지케이스/에러 경로 처리(빈 입력, 외부 실패, 타임아웃).
- [ ] LLM 출력 검증이 있는가(파싱 실패 대비).

### 아키텍처
- [ ] 의존성 방향 준수. 모듈이 다른 모듈을 직접 import 하지 않음.
- [ ] 외부 호출이 infrastructure 를 경유.
- [ ] 모듈 분리 가능성이 유지됨.

### 품질
- [ ] 네이밍/스타일/타입 규칙 준수.
- [ ] 테스트가 의미 있고 결정적.
- [ ] 비밀키/프롬프트 하드코딩 없음.
- [ ] 불필요한 복잡성/중복 없음.

## 산출물
- 리뷰 노트 → [../reviews/](../reviews/) (`review-NNN.md`, 템플릿: [../templates/review.md]).
- 발견된 이슈는 수정하거나 후속 태스크로 등록.

- [ ] Blocking 이슈 0.
