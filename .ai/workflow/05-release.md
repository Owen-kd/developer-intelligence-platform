# 05 — Release (반영)

> 목적: 검증된 변경을 안전하게 반영하고, 배운 것을 기록한다.

## 입력
- 리뷰 통과한 변경

## AI가 할 일
1. **최종 점검** — `ruff` / `mypy` / `pytest` 재실행. 마이그레이션이 있으면 적용 순서 확인.
2. **머지** — 작은 논리 단위 커밋, 명확한 메시지. 필요 시 PR.
3. **문서 갱신**
   - [../context/current-task.md] — 완료 항목 체크, 다음 작업.
   - [../context/roadmap.md] — Phase 진행 반영(해당 시).
   - 새 지식은 [../knowledge/], 결정은 [../decisions/].
4. **회고 한 줄** — 무엇이 잘/안 됐는지, 다음에 바꿀 것.

## 배포(향후 환경 갖춰지면)
- 마이그레이션은 파이프라인에서 순서대로.
- `APP_DEBUG=false`, 시크릿은 시크릿 매니저.
- 배포 후 `/health` 확인.

## 게이트
- [ ] 모든 체크 통과.
- [ ] 문서/로드맵/태스크 갱신 완료.
- [ ] 롤백 방법이 명확(마이그레이션 역방향 등).
