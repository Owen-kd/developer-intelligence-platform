# 01 — Discovery (요구사항 분석)

> 목적: 무엇을 왜 만드는지 합의한다. **코드를 쓰기 전에** 끝낸다.

## 입력
- 사용자 요청 / 이슈 / 아이디어

## AI가 할 일
1. **문제 재진술** — 요청을 한 문단으로 다시 쓴다. "누가, 무엇을, 왜"를 명확히.
2. **범위 정의** — 이번에 하는 것 / 안 하는 것(Non-goals)을 나눈다.
3. **제약 확인** — 아키텍처 원칙([../core/architecture-principles.md]), 기존 모듈, 데이터 소스.
4. **불확실성 목록** — 모르는 것/가정을 나열하고, 필요하면 **질문한다**(추측 금지).
5. **성공 기준** — "무엇이 되면 done인가"를 검증 가능하게 적는다.

## 산출물
- 짧은 Discovery 노트(문제/범위/비목표/가정/성공기준).
- 큰 작업이면 [../tasks/](../tasks/) 에 태스크로 기록.
