# Playbook — Release (릴리즈 판단)

> 릴리즈 준비 상태를 점검하고 릴리즈 노트를 만든다.

## 입력
- 릴리즈 범위(태그/브랜치/커밋 범위)

## 절차
### 1. 변경 집계
- 포함된 커밋/PR/이슈 목록, 사용자 관점 변화 요약.

### 2. 리스크 스캔
- 미해결 Blocking 리뷰, 마이그레이션, 호환성 파괴 API 변경.

### 3. 마이그레이션 확인
- `database/migrations` 적용 순서/역방향(롤백) 가능성.

### 4. 체크리스트 검증
- 테스트/린트/타입 통과, `/health` 정상, 문서 갱신.

### 5. 릴리즈 노트
- 하이라이트 / 변경 / 수정 / 주의사항(마이그레이션 등).

### 6. Go / No-Go
- 판정 + 근거. No-Go면 남은 조건을 명시.

## 규칙
- 릴리즈는 [../workflow/05-release.md] 게이트를 통과해야 한다.
- 배포 후 검증(`/health`, 핵심 지표) 항목을 노트에 포함한다.
