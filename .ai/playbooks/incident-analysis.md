# Playbook — Incident Analysis (장애 분석)

> 장애 발생 시 관련 맥락을 자동 수집·정리하여 대응/회고를 돕는다.

## 입력
- 장애 신호(알림/이슈) + 시간 범위

## 절차
### 1. 타임라인 수집
- 장애 전후 배포/커밋/설정 변경, 관련 이슈/코멘트.

### 2. 변경 상관관계
- 시간 범위 내 커밋/PR/마이그레이션과 증상 매칭 (`(:Commit)-[:FIXES|TOUCHES]`).

### 3. 영향 범위
- 영향받은 서비스/모듈/API/데이터. Impact Analysis 플레이북 재사용.

### 4. 가설 정리
- 추정 원인 후보 + 근거 + 반증. 확신도 표기.

### 5. 대응/완화
- 즉시 완화책, 롤백 후보(마이그레이션 역방향 포함).

### 6. 회고 문서
- 템플릿: [../templates/incident.md]. 타임라인/원인/조치/재발방지.

## 규칙
- 사실(로그/커밋)과 가설을 명확히 구분한다.
- 자동 수집은 infrastructure 경유, 결과는 감사 로그(`platform/audit`)에 남긴다.
